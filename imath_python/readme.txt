图片验证码的使用逻辑：浏览器--服务器：
1. 浏览器生成一个编号（解决多用户的重复性），向后端发送请求，编号,后端生成验证码图片并保存，因为验证码中有有效期，所以需要缓存在redis数据库中
2. 返回验证码图片，等待下一次请求
3. 放松短信验证码的请求，携带参数：用户填写的图片验证码，
4. 验证图片验证码的正确性，如果正确，发送短信验证码
